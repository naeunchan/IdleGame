# Apps in Toss WebView MVP Release Checklist

이 문서는 `개발견 키우기`의 MVP를 Toss WebView에 올리기 전에 확인하는 짧은 릴리즈 체크리스트입니다.

## 빌드/실행

- `yarn dev`로 로컬 개발 셸이 뜨는지 확인한다.
- `yarn build`가 성공하는지 확인한다.
- `yarn lint`, `yarn typecheck`, `yarn test`가 모두 통과하는지 확인한다.

## WebView 동작

- 세로형 화면에서 첫 진입 시 흰 화면이나 레이아웃 깨짐이 없는지 확인한다.
- safe-area 상단/하단이 잘려 보이지 않는지 확인한다.
- 스크롤이 불필요하게 튀지 않고, 버튼 탭 영역이 충분히 큰지 확인한다.
- 회전, 백그라운드 복귀, 재진입 시 화면이 정상 복원되는지 확인한다.

## 게임 플로우

- 새 저장으로 시작했을 때 1인 개발견과 초기 자원이 정상으로 보이는지 확인한다.
- 온보딩이 첫 진입에만 나타나고, 닫은 뒤에는 유지되는지 확인한다.
- 첫 고용까지의 흐름이 막히지 않는지 확인한다.
- `폭포형`, `나선형`, `애자일` 전환이 모두 동작하는지 확인한다.
- 오프라인 복구 후 자원과 진행도가 자연스럽게 이어지는지 확인한다.
- 저장이 손상되었을 때 안전한 초기화 경로로 떨어지는지 확인한다.

## MVP 수용 기준

- 시작, 저장, 복구, 고용, 프로세스 전환, 회사 확장 중 하나라도 막히면 릴리즈하지 않는다.
- 콘솔 에러, 치명적 경고, 빈 화면, 입력 먹통이 있으면 릴리즈하지 않는다.
- 작은 화면과 일반 화면 모두에서 첫 10분 플레이가 가능해야 한다.

## 참고

- 현재 브랜치 스택과 PR 흐름은 README의 `스택된 PR 구조` 섹션을 따른다.
- 구현이 바뀌면 이 체크리스트는 실제 UI 플로우와 저장 정책에 맞춰 같이 갱신한다.
